直播中的User、ChannelID、StreamID等信息都是业务侧自己定义和维护，userID，username的长度不超过255个字节，且非空。channelID的长度不超过64个字符，StreamID的长度不超过128个字符，且非0非空，在同一个时间，StreamID必须唯一，只有一个直播源结束之后，这个StreamID才能被其他的直播源重新启用。

//为了实现发布直播秒开，可以在viewDidLoad时就开始预览
    [getZegoAV_ShareInstance() setLocalViewMode:ZegoVideoViewModeScaleAspectFill];
    [getZegoAV_ShareInstance() setLocalView:self.preView];
    [getZegoAV_ShareInstance() startPreview];
// 设置回调
getZegoAV_ShareInstance().delegate = self;
// 登录频道，sdk会在onLoginChannel回调中返回登陆结果
// loginChannel这个步骤不能省略，发布直播和观看直播都要先loginchannel。
[getZegoAV_ShareInstance() loginChannel:self.liveChannel user:user];

/// @直播回调
interface ZegoLiveApiDelegateImpl () <ZegoLiveApiDelegate>
end
implementation ZegoLiveApiDelegateImpl

// channelID的长度不超过64个字符，StreamID的长度不超过128个字符，且非0非空，在同一个时间，StreamID必须是唯一的，只有一个直播源结束之后，这个StreamID才能被其他的直播源重新启用。
/// \brief 登录频道的回调
- (void)onLoginChannel:(NSString *)channel error:(uint32)err {
    if（err == 0）{
        // 登录成功，开始发布直播
        [getZegoAV_ShareInstance() startPublishingWithTitle:liveTitle streamID:streamID];
    }else{
        // 登录频道失败，提示用户
    }
}

/// \brief 发布直播成功
- (void)onPublishSucc:(NSString *)streamID channel:(NSString *)channel streamInfo:(NSDictionary *)info {
    NSLog(@"%s, stream: %@, url: %@", __func__, streamID, playUrl);
}

/// \brief 发布直播失败
/// \param err 1：正常结束，其它错误码可在集成指南最后参考
- (void)onPublishStop:(uint32)err stream:(NSString *)streamID channel:(NSString *)channel {
    NSLog(@"%s, stream: %@, err: %u", __func__, streamID, err);   
}

七、连麦直播

1. 连麦流程图

2. 连麦说明

1. 连麦情况下，多主播都必须在同一channel下才能实现连麦。ChannelID是一个直播频道（可以重复利用），建议在一个直播源结束之后，这个channelId才给其他的直播源重新启用。

2. 连麦成功后，如果两路连麦，观众需要设置两个View（setRemoteView:(RemoteViewIndex)index view:(UIView*)view） 去播放对应的两条流。
注意：这两个view的index不能一致，同时这个index必须要和startPlayStream传入的index一致。（startPlayStream:(NSString *)streamID viewIndex:(RemoteViewIndex)index）

3. 一般来说，连麦的实现是主播和该主播的观众之间进行，如上图。该图中观众（第二主播）直接startpublish一条流就可以，不需要再重新loginchannel了，因为观众已经在第一主播的channel里面了。

若主播和主播之间连麦的话，业务侧可以自己实现，只要确保两主播在同一个channel就可以。比如A主播要连麦B主播，可以让A主播stoppublish,登出原来的channel，然后登录B主播的channel去startpublish，这个过程，业务侧可以做一个过场动画，观众也需要有一个连接新主播的过场动画过程。

4. 连麦过程中，主播A连上主播B的时候，可能会出现主播B画面卡顿1s左右，这是因为连麦成功后，播放线路优化的过程，优化后能降低连麦的延迟。

1. 屏幕旋转处理方式

1.1 正常直播 屏幕旋转处理方式

//屏幕旋转时调用setDisplayRotation接口
- (void)setRotateFromInterfaceOrientation:(UIInterfaceOrientation)fromInterfaceOrientation
{
    switch (fromInterfaceOrientation) {
        case UIInterfaceOrientationPortrait:
            [getZegoAV_ShareInstance() setDisplayRotation:CAPTURE_ROTATE_0];
            break;

        case UIInterfaceOrientationPortraitUpsideDown:
            [getZegoAV_ShareInstance() setDisplayRotation:CAPTURE_ROTATE_180];
            break;

        case UIInterfaceOrientationLandscapeLeft:
            [getZegoAV_ShareInstance() setDisplayRotation:CAPTURE_ROTATE_270];
            break;

        case UIInterfaceOrientationLandscapeRight:
            [getZegoAV_ShareInstance() setDisplayRotation:CAPTURE_ROTATE_90];
            break;

        default:
            break;
    }
}

//ios8以上重载此方法
- (void)viewWillTransitionToSize:(CGSize)size withTransitionCoordinator:(id<UIViewControllerTransitionCoordinator>)coordinator
{
//    NSLog(@"viewWillTransitionToSize");
    [coordinator animateAlongsideTransition:^(id<UIViewControllerTransitionCoordinatorContext>  _Nonnull context) {
        UIInterfaceOrientation orientation = [[UIApplication sharedApplication] statusBarOrientation];
        [self setRotateFromInterfaceOrientation:orientation];
    } completion:^(id<UIViewControllerTransitionCoordinatorContext>  _Nonnull context) {

    }];

    [super viewWillTransitionToSize:size withTransitionCoordinator:coordinator];
}

2. 来电响应处理方式

 //1.电话状态监听（监听广播AVAudioSessionInterruptionNotification）

[[NSNotificationCenter defaultCenter] addObserver:self selector:@selector(audioSessionWasInterrupted:) name:AVAudioSessionInterruptionNotification object:nil];

//2.observer中实现audio session的开始和结束
- (void)audioSessionWasInterrupted:(NSNotification *)notification
{
    if ([notification.userInfo count] == 0)
    {
        return;
    }

    if (AVAudioSessionInterruptionTypeBegan == [notification.userInfo[AVAudioSessionInterruptionTypeKey] intValue])
    {
        if(self.isPublishing && self.liveType == 2){
            _publishAfterPlay = YES;
        }

        // 停止直播(主播方)
        if (self.isPublishing) {
            self.isPublishing = NO;
            [getZegoAV_ShareInstance() stopPreview];
            [getZegoAV_ShareInstance() stopPublishing];
        }

        // 停止播放（观众方）
        if([_playStreamArray count] > 0){
            for(int i = 0; i < [_playStreamArray count]; i++){
                [getZegoAV_ShareInstance() stopPlayStream:[_playStreamArray objectAtIndex:i]];
            }
        }

        // 退出房间
        [getZegoAV_ShareInstance() logoutChannel];

    }
    else if(AVAudioSessionInterruptionTypeEnded == [notification.userInfo[AVAudioSessionInterruptionTypeKey] intValue])
    {
        // 恢复发布或观看
    }
}

十一、其他API说明

API 说明
setBusinessType 设置业务类型。0 - 支持多主播连麦, 1 - 仅支持单主播直播(此时要求 liveID 与 streamID 相同)，默认为 0
setAVConfig 设置视频配置
enableNoiseSuppress 开启关闭音频采集噪声抑制
enableMic 开启关闭麦克风
enableSpeaker （声音输出）静音开关
setBuiltInSpeakerOn 手机内置扬声器开关
enableCamera 开启或关闭视频采集
takeRemoteViewSnapshot 截取观看方view 图像
takeLocalViewSnapshot 截取主播方view 图像
enableTorch 开启或关闭手电筒
enableBeautifying 设置美颜
setFilter 设置滤镜
version 获取 SDK 版本
setVideoCaptureFactory 设置外部采集模块

回调接口说明：

API 说明
onVideoSizeChanged 视频的宽度和高度发生变化后回调
onTakeRemoteViewSnapshot 观看方截屏结果回调
onTakeLocalViewSnapshot 主播方截屏结果回调
onCaptureVideoSizeChangedToWidth publish后返回视频的宽、高
