七、连麦直播

1. 连麦流程

2. 连麦说明

1. 连麦情况下，多主播都必须在同一channel下才能实现连麦。ChannelID是一个直播频道（可以重复利用），在同一个时间，channelID是唯一的，只有一个直播源结束之后，这个channelId才能被其他的直播源重新启用。

2. 连麦成功后，如果两路连麦，观众需要设置两个remoteview去播放对应的两条流。

3. 一般来说，连麦的实现是主播和该主播的观众之间进行，如上图。
若主播和主播之间连麦的话，业务侧可以自己实现，只要确保两主播在同一个channel就可以。比如A主播要连麦B主播，可以让A主播stoppublish,登出原来的channel，然后登录B主播的channel去startpublish，这个过程，业务侧可以做一个过场动画，观众也需要有一个连接新主播的过场动画过程。

4. 连麦过程中，主播A连上主播B的时候，可能会出现主播B画面卡顿1s左右，这是因为连麦成功后，播放线路优化的过程，优化后能降低连麦的延迟。

1. 屏幕旋转处理方式

1.1 正常直播 屏幕旋转处理方式
/**
 * 初始化屏幕旋转监听器，在Activity的onCreate方法中调用.
*/
protected void initRotationListener(){
    // 部分安卓手机不支持180度转屏，以致于从90度转到270度或者从270度转到90度时，系统认为界面没有发生变化，
    // 所以没有好的时机调用mZegoAVKit.setDisplayRotation来改变画面的角度，用
    //  DisplayManager.DisplayListener可以达到目的，但只支持android 17以上的系统，如果开发者有更好的办法还请告知Zego，多谢
    DisplayManager.DisplayListener mDisplayListener;
    if (Build.VERSION.SDK_INT >= Build.VERSION_CODES.JELLY_BEAN_MR1) {
          mDisplayListener = new DisplayManager.DisplayListener() {
              @Override
              public void onDisplayAdded(int displayId) {
                  Log.i("Display", "Display #" + displayId + " added.");
              }
              @Override
              public void onDisplayChanged(int displayId) {
                    // Log.i("Display", "Display #" + displayId + " changed.");
                    // 改变画面角度
                    changeRotation();
                }
                @Override
                public void onDisplayRemoved(int displayId) {
                    Log.i("Display", "Display #" + displayId + " removed.");
                }
            };

            DisplayManager displayManager = 
                        (DisplayManager) getSystemService(Context.DISPLAY_SERVICE);
            displayManager.registerDisplayListener(mDisplayListener, mHandler);
   }else {
       // android 17以下的系统通过横竖屏变化后Activity生命周期的调用来改变画面的角度
       // 这种方法不能适配前面说的问题
       changeRotation);
   }
}

protected void changeRotation() {
    switch (getWindowManager().getDefaultDisplay().getRotation()) {
         case Surface.ROTATION_0:
              mZegoAVKit.setDisplayRotation(
                  ZegoAVKitCommon.ZegoCameraCaptureRotation.Rotate_0);
              break;
         case Surface.ROTATION_90:
              mZegoAVKit.setDisplayRotation(
                  ZegoAVKitCommon.ZegoCameraCaptureRotation.Rotate_90);
              break;
         case Surface.ROTATION_180:
              mZegoAVKit.setDisplayRotation(
                  ZegoAVKitCommon.ZegoCameraCaptureRotation.Rotate_180);
              break;
         case Surface.ROTATION_270:
              mZegoAVKit.setDisplayRotation(
                  ZegoAVKitCommon.ZegoCameraCaptureRotation.Rotate_270);
              break;
        }
 }

2. 来电处理方式

 /**
    * 电话状态监听.
 */
 // 初始化来电状态, false表示没有来电
 private boolean mHaveBeenCalled = false;
 protected void initPhoneCallingListener(){
    TelephonyManager tm = (TelephonyManager)getSystemService(Service.TELEPHONY_SERVICE);
    tm.listen(new PhoneStateListener(){
       @Override
       public void onCallStateChanged(int state, String incomingNumber) {
             super.onCallStateChanged(state, incomingNumber);
             switch(state){
                 case TelephonyManager.CALL_STATE_RINGING:
                     Log.i("TestPhoneState", "响铃:来电号码"+incomingNumber);
                     if(!mHaveBeenCalled){
                         // 标记已来电
                         mHaveBeenCalled ＝ true;

                         // 来电停止发布与播放
                         if(mIsPlaying){
                            mZegoAVKit.stopPlayStream(playStreamID);
                            mZegoAVKit.setRemoteView(zegoRemoteViewIndex, null);
                         }
                         if(mIsPublishing){
                            mZegoAVKit.stopPreview();
                            mZegoAVKit.stopPublish();
                            mZegoAVKit.setLocalView(null);
                         }
                         // 退出频道
                         mZegoAVKit.logoutChannel();
                      }
                      break;
                 case TelephonyManager.CALL_STATE_IDLE:
                      Log.i("TestPhoneState", "挂断电话");
                      if（mHaveBeenCalled){
                          // 恢复来电状态为false
                          mHaveBeenCalled = false;

                         // 重新登陆频道
                         ZegoUser zegoUser = new ZegoUser(
                         PreferenceUtils.getInstance().getUserID(), 
                             PreferenceUtils.getInstance().getUserName());
                         mZegoAVKit.loginChannel(zegoUser, mLiveChannel);

                      }

                     break;
                case TelephonyManager.CALL_STATE_OFFHOOK:
                     Log.i("TestPhoneState", "接听");
                     break;
            }
       }
   }, PhoneStateListener.LISTEN_CALL_STATE);
}

十二、SDK错误码说明

在LoginChannel的回调中出现以下错误码 说明：

错误码 说明
SUCCESS = 0 channel登录成功
INPUT_ERROR = 2 参数输入错误
AUTH_FAILURE = 3 鉴权失败
APPID_INPUT_ERROR = 102 APPID输入错误

在Publish和Play的回调中出现以下错误码 说明：

错误码 说明
AVStateEnd = 1 直播正常停止
TempBroken = 2 直播临时中断
FatalError = 3 直播遇到严重的问题
CreateStreamError=4 创建直播流失败
FetchStreamError=5 获取流信息失败
NoStreamError=6 无流信息
MediaServerNetWorkError=7 媒体服务器连接失败
DNSResolveError=8 DNS 解释失败
NotLoginError = 9 未登陆channel就直接play/publish

十三、其他API说明

API 说明
setBusinessType 设置业务类型。0 - 支持多主播连麦, 1 - 仅支持单主播直播(此时要求 liveID 与 streamID 相同)，默认为 0
setAVConfig 设置视频配置
enableNoiseSuppress 开启关闭音频采集噪声抑制
enableMic 开启关闭麦克风
enableSpeaker （声音输出）静音开关
setBuiltInSpeakerOn 手机内置扬声器开关
enableCamera 开启或关闭视频采集
takeRemoteViewSnapshot 截取观看方view 图像
takeLocalViewSnapshot 截取主播方view 图像
enableTorch 开启或关闭手电筒
enableBeautifying 设置美颜
setFilter 设置滤镜
version 获取 SDK 版本
setVideoCaptureFactory 设置外部采集模块

回调接口说明：

API 说明
onVideoSizeChanged 视频的宽度和高度发生变化后回调
onTakeRemoteViewSnapshot 观看方截屏结果回调
onTakeLocalViewSnapshot 主播方截屏结果回调
onCaptureVideoSizeChangedToWidth publish后返回视频的宽、高
